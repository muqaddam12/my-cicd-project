üìñ How the Automation Works

This project demonstrates a production-grade Continuous Integration and Continuous Deployment (CI/CD) workflow. When a commit is pushed to the GitHub repository, a Webhook triggers AWS CodePipeline. The pipeline fetches the source, passes it to CodeBuild for validation using the buildspec.yml, and finally hands it to CodeDeploy. CodeDeploy communicates with the agent on the EC2 instance to update the index.html in the /var/www/html/ directory based on the appspec.yml instructions.
